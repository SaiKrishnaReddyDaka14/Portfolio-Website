Edge-Cloud AI-Powered Face Recognition System using AWS Greengrass, Lambda & ECR

A real-time AI-powered face recognition system with intelligent edge processing, cloud-based inference, and event-driven communication.

Key Highlights & Architecture:

  • Deployed a distributed edge-cloud architecture for real-time face recognition by deploying MTCNN-based face detection on an AWS IoT Greengrass Core (EC2) and FaceNet-based recognition on AWS Lambda, achieving 98.7% classification accuracy on live video frames.
  • Containerized all ML components using Docker and published to Amazon ECR, enabling lightweight and scalable deployment across edge (Greengrass) and cloud (Lambda) environments under a PaaS model optimized for event-driven execution.
  • Established a secure, low-latency messaging pipeline using MQTT 3.1.1 and TLS for edge-to-cloud communication, with PubSub and topic mapping to route base64-encoded frames from IoT clients to the Greengrass Core for pre-processing.
  • Reduced cloud workload by 40% by intelligently filtering “No-Face” inputs at the edge and sending only detected faces to the cloud for recognition, lowering Lambda invocation costs by 35% and ensuring faster response times for real-time use cases.

Technologies Used:

Python AWS Greengrass AWS Lambda AWS ECR AWS SQS MQTT PyTorch MTCNN FaceNet Edge Computing

Multi-Modal Retrieval-Augmented Generation (RAG) System with LLM for Advanced Analysis

An advanced retrieval-augmented generation pipeline that extracts and semantically indexes multi-modal content from research PDFs to answer the user queries using LLaMA-3.2-90B-Vision-Instruct model.

Key Highlights & Architecture:

  • Built a multi-modal RAG pipeline for research PDF processing, allowing users to upload documents via a web UI, which were stored in Amazon S3 and indexed in DynamoDB for fast retrieval and traceability.
  • Integrated PyMuPDF and pdfplumber to extract raw text, structured tables, and embedded images; applied LLaMA-3.2-90B-Vision-Instruct to generate descriptive captions for all visual elements.
  • Applied preprocessing steps to unify all modalities into text, segmented them into semantically meaningful chunks, and embedded them using SentenceTransformers for dense vector representation.
  • Engineered a vector-based retrieval mechanism using FAISS to match user queries with the most relevant content chunks and passed them to LLaMA-3.2-90B for context-aware answer generation.
  • Enabled fast, automated semantic search over academic documents, significantly improving answer quality and reducing manual effort in technical literature analysis.
